Ejercicio 10

Objetivo

Crear una función que reciba el nombre de un aventurero y su nivel, y devuelva un mensaje de bienvenida personalizado. Llamarla desde el Event Graph con 3 aventureros distintos.

Resultado log y Pantalla

¡Bienvenido Nombre, nivel 5! Toma asiento aventurero.
¡Bienvenido Nombre, nivel 8! Toma asiento aventurero.
¡Bienvenido Nombre, nivel 3! Toma asiento aventurero.

Resultado al compilar y ejecutar el BP

Procedimiento
  1. En el panel My Blueprint, buscar la sección Functions, clickear el + y nombrar la función ConstruirSaludo.
  2. Se abre automáticamente el gráfico de la función con un nodo Entry y otro de Return (al establecer un output).
  3. Con la función seleccionada, en el panel Details configurar los Inputs y el Output:
    • Input: NombreAventurero (String)
    • Input: Nivel (Integer)
    • Output: Mensaje (String)
  4. Dentro del gráfico de la función, arrastrar los pins NombreAventurero y Nivel desde el nodo Entry para usarlos en la construcción del mensaje. Armar el mensaje "¡Bienvenido NombreAventurero, nivel Nivel! Toma asiento aventurero." usando nodos Append encadenados:
    • Primer Append: pin A → "¡Bienvenido ", pin B → NombreAventurero.
    • Segundo Append: pin A → salida del anterior, pin B → ", nivel ".
    • Tercer Append: pin A → salida del anterior, pin B → ToString del pin Nivel.
    • Cuarto Append: pin A → salida del anterior, pin B → "! Toma asiento aventurero.".
    • Conectar la salida del último Append al pin Mensaje del Return Node.
  5. Volver al Event Graph y conectar Event BeginPlay a tres llamadas encadenadas al nodo ConstruirSaludo, cada una seguida de un Print String:
    • Llamada 1:
      • Pin NombreAventurero → escribir un nombre.
      • Pin Nivel → 5.
      • Salida Mensaje → pin In String de un Print String.
    • Llamada 2:
      • Pin NombreAventurero → escribir un nombre.
      • Pin Nivel → 8.
      • Salida Mensaje → pin In String de un Print String.
    • Llamada 3:
      • Pin NombreAventurero → escribir un nombre.
      • Pin Nivel → 3.
      • Salida Mensaje → pin In String de un Print String.
  6. Encadenar los tres Print String uno tras otro por el pin de ejecución.
  7. Compilar, guardar, arrastrar a la escena y presionar Play.

Ejercicio 11

Objetivo

Un guardia permite el acceso al pueblo solo si el viajero tiene al menos 10 monedas de peaje. Usar un comparador numérico y un Branch para imprimir si puede entrar o es rechazado

Resultado log y Pantalla

===== CONTROL DE ACCESO AL PUEBLO =====
Lo siento Kael, necesitás al menos 10 monedas. Tenés: 7

Resultado al compilar y ejecutar el BP

Procedimiento
  1. Crear variable Monedas (Integer) con valor default 7.
  2. Crear variable NombreViajero (String) con valor default "Kael".
  3. En el Event Graph, conectar Event BeginPlay a un nodo Print String con el mensaje "===== CONTROL DE ACCESO AL PUEBLO =====".
  4. Conectar el Print String a un nodo Branch. Armar la condición Monedas >= 10:
    • Arrastrar GET Monedas y conectarlo a un nodo >= (Integer) con valor 10.
    • Conectar la salida Boolean del >= al pin Condition del Branch.
  5. Desde el nodo Branch:
    • TRUE → Print String usando Append: "Bienvenido " + NombreViajero + "! Podés entrar al pueblo." en color verde.
    • FALSE → Print String usando Append: "Lo siento " + NombreViajero + ", necesitás al menos 10 monedas. Tenés: " + Monedas en color rojo.
  6. Compilar, guardar, arrastrar a la escena y presionar Play.

Ejercicio 12

Objetivo

Un cofre solo se abre si el jugador tiene la llave correcta Y no está maldito. Usar dos variables booleanas combinadas con AND para determinar si el cofre se abre o permanece cerrado.

Resultado log y Pantalla

===== INTENTO DE APERTURA DEL COFRE =====
¡El cofre se abre! Aragorn encontró el tesoro.

Resultado al compilar y ejecutar el BP

Procedimiento
  1. Crear variable TieneLlave (Boolean) con valor default true.
  2. Crear variable EstaMaldito (Boolean) con valor default false.
  3. Crear variable NombreJugador (String) con valor default "Aragorn".
  4. En el Event Graph, conectar Event BeginPlay a un nodo Print String con el mensaje "===== INTENTO DE APERTURA DEL COFRE =====".
  5. Conectar el Print String a un nodo Branch. Armar la condición TieneLlave AND NOT EstaMaldito:
    • Arrastrar GET TieneLlave al pin A de un nodo AND.
    • Arrastrar GET EstaMaldito a un nodo NOT y conectar su salida al pin B del mismo AND.
    • Conectar la salida del AND al pin Condition del Branch.
  6. Desde el nodo Branch principal:
    • TRUE → Print String usando Append: "¡El cofre se abre! " + NombreJugador + " encontró el tesoro." en color amarillo.
    • FALSE → conectar a un segundo nodo Branch con condición NOT TieneLlave:
      • GET TieneLlave → nodo NOT → pin Condition del segundo Branch.
      • TRUE → Print String: "No tenés la llave. El cofre permanece cerrado." en color rojo.
      • FALSE → Print String: "Estás maldito. No podés abrir el cofre." en color rojo oscuro.
  7. Compilar, guardar, arrastrar a la escena y presionar Play.

Ejercicio 13

Objetivo

Un troll bloquea un puente y solo deja pasar a quien sea un guerrero de alto nivel, O tenga un Pase real, pero NUNCA a alguien que esté maldito, sin importar las otras condiciones.

Resultado log y Pantalla

===== EL PUENTE DEL TROLL =====
El troll gruñe... pero deja pasar a Finn. Su nivel de 75 impone respeto.

Resultado al compilar y ejecutar el BP

Procedimiento
  1. Crear variable NombreViajero (String) con valor default "Finn".
  2. Crear variable Nivel (Integer) con valor default 75.
  3. Crear variable TienePase (Boolean) con valor default false.
  4. Crear variable EstaMaldito (Boolean) con valor default false.
  5. En el Event Graph, conectar Event BeginPlay a un nodo Print String con el mensaje "===== EL PUENTE DEL TROLL =====".
  6. Conectar el Print String a un primer nodo Branch con condición NOT EstaMaldito:
    • Arrastrar GET EstaMaldito a un nodo NOT y conectar su salida al pin Condition.
    • FALSE → Print String usando Append: "¡Atrás, " + NombreViajero + "! El troll huele la maldición. Nadie maldito cruza este puente." en color rojo.
  7. De la rama TRUE del primer Branch, conectar un segundo Branch con condición Nivel >= 70 OR TienePase:
    • Arrastrar GET Nivel a un nodo >= (Integer) con valor 70 → pin A de un nodo OR.
    • Arrastrar GET TienePase → pin B del mismo OR.
    • Conectar la salida del OR al pin Condition del segundo Branch.
    • FALSE → Print String usando Append: "¡ALTO, " + NombreViajero + "! Necesitás ser nivel 70 o tener un Pase real." en color naranja.
  8. De la rama TRUE del segundo Branch, conectar un tercer Branch con condición Nivel >= 70 para personalizar el mensaje:
    • TRUE → Print String usando Append: "El troll gruñe... pero deja pasar a " + NombreViajero + ". Su nivel de " + Nivel + " impone respeto." en color verde.
    • FALSE → Print String usando Append: "El troll examina el Pase de " + NombreViajero + " y se hace a un lado a regañadientes." en color verde.
  9. Compilar, guardar, arrastrar a la escena y presionar Play.

Ejercicio 14

Objetivo

Simular una tienda que ofrece distintos descuentos según la combinación de condiciones del cliente: si es miembro del gremio, si su nivel supera cierto rango, y si tiene reputación suficiente. Cada combinación da un resultado distinto.

Resultado log y Pantalla

===== TIENDA DEL ALQUIMISTA =====
Cliente: Mira
¡Descuento del 50%! Sos miembro del gremio, nivel alto y gran reputación.

Resultado al compilar y ejecutar el BP

Procedimiento
  1. Crear variable NombreCliente (String) con valor default "Mira".
  2. Crear variable EsMiembroGremio (Boolean) con valor default true.
  3. Crear variable Nivel (Integer) con valor default 55.
  4. Crear variable Reputacion (Integer) con valor default 80.
  5. En el Event Graph, conectar Event BeginPlay a dos nodos Print String encadenados:
    • "===== TIENDA DEL ALQUIMISTA ====="
    • "Cliente: " + NombreCliente.
  6. Conectar al último Print String un primer nodo Branch con condición GET EsMiembroGremio directamente en el pin Condition.
  7. Rama TRUE (es miembro del gremio) → segundo Branch con condición Nivel >= 50 AND Reputacion >= 80:
    • GET Nivel → >= (Integer) con valor 50 → pin A de un AND.
    • GET Reputacion → >= (Integer) con valor 80 → pin B del mismo AND.
    • TRUE → Print String: "¡Descuento del 50%! Sos miembro del gremio, nivel alto y gran reputación." en color amarillo.
    • FALSE → tercer Branch con condición Nivel >= 50 OR Reputacion >= 80:
      • GET Nivel → >= (Integer) con valor 50 → pin A de un OR.
      • GET Reputacion → >= (Integer) con valor 80 → pin B del mismo OR.
      • TRUE → Print String: "¡Descuento del 30%! Sos miembro del gremio con buen nivel o reputación." en color verde.
      • FALSE → Print String: "Sin descuento. Sos miembro del gremio pero tu nivel y reputación son bajos." en color naranja.
  8. Rama FALSE (no es miembro del gremio) → cuarto Branch con condición Reputacion >= 70:
    • GET Reputacion → >= (Integer) con valor 70 → pin Condition.
    • TRUE → Print String: "Descuento del 15%. No sos miembro del gremio pero tu reputación te respalda." en color verde.
    • FALSE → Print String usando Append: "¡Fuera de mi tienda, " + NombreCliente + "! Sin gremio y sin reputación no sos bienvenido." en color rojo.
  9. Compilar, guardar, arrastrar a la escena y presionar Play.

Ejercicio 15

Objetivo

Clasificar a un campeón en una de cuatro categorías de torneo (Bronce, Plata, Oro, Legendario) o descalificarlo, encapsulando la lógica de evaluación en una función reutilizable que recibe los stats como inputs, combina condiciones con AND y OR, y devuelve la categoría correspondiente como output.

Resultado log y Pantalla

===== EL TORNEO DE LOS REINOS =====
Aldric -> Categoria: ORO!

Resultado al compilar y ejecutar el BP

Procedimiento
  1. Crear variable NombreCampeon (String) con valor default "Aldric".
  2. Crear variable Fuerza (Integer) con valor default 85.
  3. Crear variable Agilidad (Integer) con valor default 75.
  4. Crear variable Magia (Integer) con valor default 60.
  5. Crear variable Reputacion (Integer) con valor default 90.
  6. Crear variable EstaDescalificado (Boolean) con valor default false.
  7. En el panel My Blueprint, sección Functions, clickeár el + y nombrar la función ObtenerCategoria.
  8. Con la función seleccionada, en el panel Details configurar los Inputs y el Output:
    • Input: Fuerza (Integer)
    • Input: Agilidad (Integer)
    • Input: Magia (Integer)
    • Input: Reputacion (Integer)
    • Input: EstaDescalificado (Boolean)
    • Output: Categoria (String)
  9. Dentro del gráfico de la función, armar la lógica de clasificación en cascada desde el nodo ObtenerCategoria (Entry):
    • Primer Branch con condición GET EstaDescalificado directamente en el pin Condition:
      • TRUE → Return Node con Categoria = "DESCALIFICADO".
    • Rama FALSE → segundo Branch con condición Fuerza >= 90 AND Agilidad >= 80 AND Magia >= 70 AND Reputacion >= 90:
      • Encadenar tres nodos AND conectando los cuatro comparadores >= (Integer) en secuencia.
      • TRUE → Return Node con Categoria = "LEGENDARIO".
    • Rama FALSE → tercer Branch con condición (Fuerza >= 75 AND Agilidad >= 70) AND (Magia >= 60 OR Reputacion >= 80):
      • Primer AND: Fuerza >= 75 y Agilidad >= 70.
      • OR: Magia >= 60 y Reputacion >= 80.
      • Segundo AND: salida del primer AND y salida del OR.
      • TRUE → Return Node con Categoria = "ORO".
    • Rama FALSE → cuarto Branch con condición (Fuerza >= 60 OR Agilidad >= 60) AND Reputacion >= 50:
      • OR: Fuerza >= 60 y Agilidad >= 60.
      • AND: salida del OR y Reputacion >= 50.
      • TRUE → Return Node con Categoria = "PLATA".
    • Rama FALSE → quinto Branch con condición Fuerza >= 40 OR Agilidad >= 40 OR Magia >= 40:
      • Encadenar dos nodos OR con los tres comparadores >= (Integer).
      • TRUE → Return Node con Categoria = "BRONCE".
      • FALSE → Return Node con Categoria = "SIN CLASIFICAR".
  10. Volver al Event Graph y conectar Event BeginPlay a un Print String con el mensaje "===== EL TORNEO DE LOS REINOS =====".
  11. Del Print String, conectar el nodo ObtenerCategoria y asignar cada pin de input:
    • Pin Fuerza → GET Fuerza
    • Pin Agilidad → GET Agilidad
    • Pin Magia → GET Magia
    • Pin Reputacion → GET Reputacion
    • Pin EstaDescalificado → GET EstaDescalificado
  12. De la salida Categoria de la función, armar un Append en cadena:
    • Primer Append: pin A → GET NombreCampeon, pin B → " → Categoría: ".
    • Segundo Append: pin A → salida del primer Append, pin B → salida Categoria.
  13. Conectar la salida del último Append al pin In String de un Print String final.
  14. Compilar, guardar, arrastrar a la escena y presionar Play.
